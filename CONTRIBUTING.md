# Contributing to AI Money Converter

Thank you for your interest in contributing to the **AI Money Converter — VB.NET Desktop Application**! Contributions are welcome and help improve the project for everyone.

As a cybersecurity-focused project, I maintain high standards for security, code quality, and professional documentation.

## How to Contribute

### 1. Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/ISMEG-ZAKARIA/ai-money-converter-vbnet/issues) to report bugs.
- Provide a clear title and description, including steps to reproduce the issue.
- If the bug is security-related, please refer to [SECURITY.md](SECURITY.md) for responsible disclosure.

### 2. Feature Requests
- Open an issue with the "feature request" label.
- Explain why the feature is needed and how it benefits the users.

### 3. Pull Requests
1. **Fork** the repository.
2. **Clone** your fork locally.
3. **Create a branch** for your change (e.g., `git checkout -b feat/my-new-feature` or `git checkout -b fix/bug-description`).
4. **Implement your changes**, ensuring you follow the existing coding style and project structure.
5. **Test your changes** thoroughly.
6. **Commit** with clear, descriptive messages following the [Conventional Commits](https://www.conventionalcommits.org/) specification (e.g., `feat: add support for cryptocurrency conversion`).
7. **Push** your branch to your fork.
8. **Open a Pull Request** against the `main` branch of this repository.

### Coding Guidelines
- Follow VB.NET naming conventions (PascalCase for classes, methods, and properties).
- Keep code clean, readable, and well-commented.
- Ensure all new features include appropriate error handling and input validation.
- Do not commit any API keys, secrets, or environment-specific configurations.

## Development Environment
- **IDE**: Visual Studio 2022
- **Framework**: .NET 8.0
- **NuGet Packages**: `Newtonsoft.Json`

## Community
If you have questions or want to discuss improvements, feel free to open a discussion or contact the maintainer directly.

Happy coding!
